Development

3D Nonogram is my current solo project — players create their own voxel-based levels and share them with others.

I started by building a level editor with a save and load system that serializes to JSON for easy backend storage.

Then I set up the server. It runs on a Raspberry Pi 5 using Python Flask. To avoid exposing my public IP or relying on port forwarding, I used a reverse proxy through ngrok — a safer way to host and test the server while learning more about server management and Unity integration.

Explanation

The script I chose to highlight is the core of 3D Nonogram's level editor — it lets players create and edit voxel-based puzzles directly inside the game. When active, it:

  • Detects voxel interactions using an optimized raycast system (Physics.RaycastNonAlloc) to avoid memory allocations
  • Handles different editing modes — placing, erasing, and painting voxels based on player input
  • Maintains undo/redo history through a custom command system, letting players revert or reapply actions
  • Synchronizes voxel data with a GridSaveData structure for smooth saving, loading, and uploading of custom levels
  • Shows a transparent voxel that follows the player's cursor, previewing where the next block will be placed

For data management, the editor serializes all voxel information into JSON files for easy storage and server integration — letting 3D Nonogram support full in-game level creation, editing, and sharing without external tools.
