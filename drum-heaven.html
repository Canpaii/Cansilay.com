Drum Heaven

Drum Heaven was my first VR project, built in 8 weeks for the Meta Quest 2/3 alongside a 3D artist. I worked as programmer and scrum master, which meant balancing technical implementation with project coordination.

Development

This project taught me the importance of using time efficiently. It was my first time bringing VR development, rhythm gameplay, and agile management together into one project.

With only 8 weeks to work, I gained valuable experience in rapid prototyping and agile iteration, learning how to adjust designs quickly when something wasn't working.

To make the drumming experience more authentic, I integrated a physical kick pedal into the game. I implemented input handling so the pedal could be used as the bass drum, working alongside the VR controller inputs.

We wanted to add a drum-roll mechanic that would behave like a hold note in other rhythm games, but I wasn't able to finish it in time, so it didn't make it into the final build.

I also learned to import and use an external library inside a Unity project — I used the DryWetMIDI library to write an algorithm that generates a beatmap directly from an imported MIDI file.

Explanation

The script I chose to highlight is the core of Drum Heaven's beatmap system. Its job is to parse a MIDI file and translate it into playable drum notes in the game. When it runs, it:

  • Parses the MIDI file and filters out the drum-specific notes
  • Places each note at its corresponding drum piece in the game, if it exists
  • Handles the lead-in delay, so notes spawn before the music begins and gameplay feels synchronized

For asset management I used StreamingAssets to reference the MIDI files. That worked well for our PC build, but caused issues when we had to build to .apk for standalone use. The system can technically process any MIDI file — if it doesn't contain drum notes, no notes spawn.
